PRÁCTICA N2 - GIT,GITHUB Y HTML
  1. ¿Qué es una rama (branch) en Git y para qué sirve?
    Una rama branch en git es como hacer una copia de tu trabajo para poder hacer cambios sin arruinar el trabajo original
  2. ¿Cómo se crea y se cambia entre ramas en Git?
    3. Para crear una rama se usa el comando git branch nombre de la rama,y para cambiar entre ramas se usa el comando git checkout nombre de la rama
  3. ¿Cuál es la diferencia entre la rama main y otras ramas?
    4. La diferencia es que la rama main es el proyecto principal y las otras ramas serian copiar para probar cosas nuevas diferentes al proyecto original
  4. ¿Cómo se combinan ramas en Git (merge)?
    5. Para combinar ramas en git primero se va a la rama principal con git checkout main luego se une a la otra rama con git merge nombre de la rama siendo asi que podemos unir los cambios de ambas ramas
  5. ¿Qué problemas pueden ocurrir al trabajar con ramas (conflictos) y cómo se resuelven?
    6. Los problemas que pueden surgir al trabajar con ramas serian los choques entre varios cambios ,o cuando dos personas trabajan en un mismo archivo y realizan cambios, para poder resolver estos conflictos se tendria que abrir el archivo en coflicto editandolo y diciendo que cambio dejar ejecutandolo con git add . y git commit
